# ADR 0004 — UX 재설계: 결정 투명성과 사용자 통제

- 상태: Accepted
- 날짜: 2026 본선
- 관련: MAS 라운드(devils-debate), design-director(decision-room), warm-editorial-ui

## 맥락 (사용자 피드백)

심사 직전 사용자 피드백:
1. UX/UI가 불편하다(결과가 카드 벽, 위계 없음).
2. "입력 순서대로 시간표가 짜이는 것 아니냐"는 **로직 불신**.
3. 컨셉에 맞는 색/배치로 재정의 필요.

조사 결과 ②는 실제 결함이었다. 초기 fallback은 `inferVerdict(line, idx)`가 입력 위치(idx)에
강하게 의존했고, 타임라인은 **고정 09:00 슬롯**을 순서대로 붙였다(오후에 써도 09:00 표시).
SDK 모드는 재정렬했지만, 사용자에게 "왜 이 순서인지"가 보이지 않아 두 모드 모두 불신을 샀다.

## 결정

MAS 라운드(decision-room 메타포)로 UX를 재정의:

1. **투명성**: 응답에 `orderRationale`("입력 순서가 아니라 무엇을 기준으로 정렬했는지") 추가.
   화면 상단 배너로 항상 노출. SDK 프롬프트에 "입력 순서를 따르지 말라 + 현재 시각" 명시.
2. **정렬 고정**: `normalizePlan`/`fallbackPlan`이 verdict 우선순위로 안정 정렬하고 `rank` 부여.
   fallback의 `inferVerdict`를 위치 의존에서 **키워드(마감/회신/언젠가 등) 신호 우선**으로 변경.
3. **현재 시각 기준 타임라인**: 고정 09:00 슬롯 제거. 호스트 TZ와 무관하게 `nowKST()`(Intl/Asia-Seoul)로
   '지금' 계산 후 그 이후로 배치. (Azure는 UTC 호스트라 코드에서 KST 보정)
4. **히어로 "지금, 이거 하나"**: 단일 다음 행동을 무대 중앙에 + 근거 + 인라인 초안.
5. **사용자 통제**: 각 결정에 `위로`/`지금으로` 버튼. 히어로 교체 시 작업물 재생성 유도. "왜"는 펼치기.
6. **색 위계(Warm Editorial)**: 단일 레드는 DO_NOW/Primary에만. SCHEDULE=인디고, DELEGATE=그린,
   DEFER=뉴트럴, DROP=흐림+취소선. verdict 의미를 색으로 구분.

## 결과 (실측)

- 단위 smoke 28 PASS(입력순서 비의존·현재시각·고정슬롯 없음 검증 포함).
- 입력 순서를 뒤섞어도(운동/사이드 먼저 입력) 보고서(마감)=1위, 사이드=DROP으로 재정렬 + 배너 설명.
- 프로덕션 타임라인 T0가 현재 KST(예: 14:55)로 표시. (`img/local-redesign.png`)

  히어로가 바뀌면 해당 작업물만 `/api/assist`로 재생성.
