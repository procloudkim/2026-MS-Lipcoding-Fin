# ADR 0001 — 브랜치 전략과 Azure 배포 smoke 우선

- 상태: Accepted
- 날짜: 2026 본선
- 관련 제약: Azure 배포(P0), 7인 AI 심사 친화 git

## 맥락

대회는 (1) Copilot SDK 실제 사용과 (2) Azure 배포를 P0로 요구한다. 배포는 인증·런타임·
리전·SKU 등 변수로 인해 막판에 실패하기 쉬운 단일 실패점이다. 또한 7명의 AI agent 심사위원이
git 이력과 문서로 평가하므로, 의사결정이 추적 가능해야 한다.

## 결정

1. **배포 경로를 가장 먼저 증명한다.** 앱 기능 이전에 정적 HTML을 Azure App Service에 올려
   public URL + 브라우저 smoke로 배포 파이프라인을 검증한다. (`test` 브랜치)
2. **브랜치 분리**
   - `main`: 환경 구성/기초.
   - `천하제일-립코딩-2026-참전`: 본선 구현.
   - `test`: 배포 smoke 실험.
3. **평가 친화 문서화**: ADR + `docs/evaluation/AI_JUDGE_BRIEF.md`로 결정/증거를 명시한다.

## 결과

- 정적 배포 smoke 선검증으로 본 배포의 리스크를 사전 제거.
- 구현 브랜치에서 기능 + SDK 증거 + 배포를 순차 진행.

## 대안

- 기능 완성 후 한 번에 배포: 막판 배포 실패 리스크가 커서 기각.
