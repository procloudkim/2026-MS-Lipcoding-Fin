# ADR 0003 — 제품 피벗: "하루 정리" → "하루 오토파일럿"

- 상태: Accepted
- 날짜: 2026 본선
- 관련: 생산성 정의, 차별화, SDK 활용도

## 맥락

초기 MVP "하루 정리"는 *텍스트 입력 → 우선순위 Top3/분류/내일* 패턴이었다. 문제점:

1. **커모디티**: "붙여넣기 → 요약/리스트"는 흔한 패턴으로 차별점이 약하다.
2. **SDK 저활용**: Copilot SDK를 텍스트 포맷터로만 사용 — 일반 LLM API로도 가능.
3. **증폭 부재**: 정리만으로는 "결정 비용 감소·실행·마무리"라는 생산성 본질에 약하게 닿는다.

## 결정

제품을 **결정 대행 에이전트 "하루 오토파일럿"**으로 격상한다.

- 조언이 아니라 **결정**: 각 항목에 DO_NOW/SCHEDULE/DEFER/DROP/DELEGATE를 단호히 부여(이유 포함).
- **결정된 하루** 타임라인 제시(시간블록).
- 1순위 작업의 **첫 작업물(초안)** 자동 생성 — 에이전트가 실제 산출물을 만든다.
- 각 작업 `시작 도와줘`로 추가 작업물 생성(멀티스텝 에이전트 = SDK 강점).
- **증폭 미터**: 대행 결정 수 · 생성 초안 수 · 절약 추정 시간.

근거(생산성 정의): 생산성 = 결정 비용↓ + 실행 + 마무리. (`prep/llm-wiki/00-event-brief.md`)

## 결과

- 한 화면에서 "쏟아내기 → 결정 → 작업물"이 완결되는 데모.
- SDK의 에이전트/멀티스텝 특성이 제품의 핵심 가치로 직결.
- 스키마: `{ headline, decisions[], timeline[], firstArtifact, amplify }`.

## 대안

- 정리형 유지 + 기능 추가: 차별화 한계로 기각.
- 장기 목표 OS / 작업 실행 워크벤치: 범위 과대 → 후속 확장 여지로 보류.
