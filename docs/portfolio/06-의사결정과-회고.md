## 1. 핵심 설계 결정 (ADR 요약)

전체 기록: [`docs/adr/`](../adr/) — 5건.

| ADR | 결정 | 이유 |
| --- | --- | --- |
| 0001 | 배포 경로를 **가장 먼저** 검증(정적 HTML smoke) | 배포는 막판 실패가 잦은 단일 실패점 → 선제거 |
| 0002 | Copilot SDK를 **토큰 인증 + 런타임 자동탐색**으로 서버 구동 | 클라우드에서 대화식 로그인 불가 → 토큰으로 비대화식 인증 |

| 0004 | **투명성 UX**(orderRationale) + 현재시각 타임라인 + 사용자 통제 | "입력 순서대로 아니냐"는 불신 제거 |
| 0005 | 보안 하드닝(helmet·에러 마스킹·권한 축소·진단 라우트 제거) | 배포 전 정보 노출·프롬프트 주입 면 차단 |

## 3. 한계와 다음 개선 (정직한 회고)

심사가 지적한 **점수를 가른 두 약점** — 둘 다 "사후 패치"가 아닌 **초기 설계** 영역이었다.

### 3-1. Azure AI & 클라우드 통합 (3/5) — 가장 큰 레버
- **현재**: AI 계층이 GitHub Copilot 런타임. Azure App Service에 배포는 됐으나 **Azure OpenAI/Foundry**는 미사용.
- **부재**: IaC(`infra/main.bicep`), 배포 파이프라인(`.github/workflows/deploy.yml`), Key Vault, Managed Identity, Application Insights.
- **다음**: Azure OpenAI 폴백 경로 추가 + bicep + GitHub Actions + Key Vault(토큰) + App Insights(관측성). → 3→4~5.

### 3-2. 혁신/독창성 (3/5)
- **현재**: 단일 프롬프트 기반 계획 생성 + 보조 초안(2 엔드포인트).
- **다음**: 실제 **function tool 호출**로 다단계 에이전트 루프 형성(예: `commit_today_plan` 도구). → SDK·혁신 동시 상승.

### 3-3. 기타 보강점
- 사용자-facing **스트리밍**(현재는 서버 버퍼링 후 일괄 반환).
- API/브라우저 **통합 테스트** 확대(현재는 `lib.js` 단위 smoke 중심).
- 개인정보 처리·보존 **고지**, 레이트 리밋.
- 결정 펼치기의 키보드 접근성(현재 `div` 클릭 의존).

## 6. 재현 방법 (포트폴리오 시연용)

```bash
git clone https://github.com/procloudkim/2026-MS-Lipcoding-Fin
cd 2026-MS-Lipcoding-Fin
npm install
npm run smoke      # 28 단위 테스트 PASS (네트워크 불필요)
npm start          # http://localhost:3000 — 폴백 모드로 즉시 동작

# 실제 SDK 응답까지 재현(선택):
#   $env:COPILOT_GH_TOKEN = (gh auth token)
#   npm start  → source: "copilot-sdk"
```

> 라이브 Azure URL은 대회 종료 후 **비용 절감을 위해 리소스를 삭제**했다. 위 명령으로 동일 동작을 로컬에서 완전히 재현할 수 있다.
