# 01. 기술 스택 (MECE)

> 계층(Layer)별로 **중복 없이(Mutually Exclusive)**, **빠짐 없이(Collectively Exhaustive)** 정리한다.
> 모든 항목은 실제 `package.json` / 소스 코드 기준이며, 추정·미사용 라이브러리는 포함하지 않는다.

---

## 1. 계층 한눈에 보기

```
┌─────────────────────────────────────────────────────────────┐
│ L1. 프레젠테이션 (브라우저)                                  │
│     Vanilla JS(ESM) · HTML · CSS · Pretendard · localStorage  │
├─────────────────────────────────────────────────────────────┤
│ L2. 애플리케이션 (Node 런타임)                                │
│     Express 4 · helmet · 정적 서빙 · REST(JSON)               │
├─────────────────────────────────────────────────────────────┤
│ L3. 도메인 코어 (네트워크 비의존)                             │
│     lib.js — 프롬프트 빌드 · 파서 · 정규화/정렬 · 폴백 엔진   │
├─────────────────────────────────────────────────────────────┤
│ L4. AI 통합 (에이전트)                                        │
│     @github/copilot-sdk · @github/copilot(런타임) · 토큰 인증  │
├─────────────────────────────────────────────────────────────┤
│ L5. 인프라 / 배포                                             │
│     Azure App Service(Linux·B1) · Oryx 빌드 · App Settings     │
├─────────────────────────────────────────────────────────────┤
│ L6. 품질 / 운영                                               │
│     node:test 스타일 smoke · 자체 심사기(judge/) · ADR        │
└─────────────────────────────────────────────────────────────┘
```

---

## 2. 계층별 상세

### L1. 프레젠테이션 계층 (Client)

| 기술 | 선택 이유 | 근거 파일 |
| --- | --- | --- |
| **Vanilla JS (ES Modules)** | 빌드 스텝 0, 의존성 0 → 배포 단순·콜드스타트 최소화 | `public/app.js` |
| **HTML5 + 시맨틱 마크업** | `lang="ko"`, `aria-live`/`aria-label`로 접근성 확보 | `public/index.html` |
| **CSS Custom Properties** | Warm Editorial 디자인 토큰(단일 레드 액센트)·반응형 | `public/styles.css` |
| **Pretendard (CDN)** | 한국어 가독성 최적 가변 폰트 | jsdelivr CDN |
| **localStorage** | 새로고침 후 마지막 결과 복원(서버 무상태 유지) | `app.js` `STORAGE_KEY` |
| **Fetch + AbortController** | 요청 취소(긴 응답 중단) UX | `app.js` `runPlan()` |

> **무상태(stateless) 원칙**: 사용자 데이터는 서버에 저장하지 않고 브라우저 `localStorage`에만 둔다. → 개인정보 노출면 최소화, 서버 스케일 단순.

### L2. 애플리케이션 계층 (Server)

| 기술 | 역할 | 근거 |
| --- | --- | --- |
| **Node.js ≥ 20.19 (배포 24 LTS)** | 런타임 | `package.json` engines |
| **Express 4** | 정적 서빙 + REST 라우팅 | `server.js` |
| **helmet 7** | 보안 헤더(CSP·HSTS·X-Frame-Options 등) | `server.js` `app.use(helmet(...))` |
| **express.json({ limit: "256kb" })** | 본문 크기 제한(DoS 완화) | `server.js` |

### L3. 도메인 코어 (`lib.js`) — 네트워크 비의존

> AI가 없어도 **순수 함수**만으로 동작/테스트 가능한 핵심 로직. 이 분리가 단위 테스트(28 PASS)와 폴백을 가능하게 한다.

| 함수 | 책임 |
| --- | --- |
| `buildPlanPrompt(context, now)` | 한국어 역할·현재시각·재정렬 원칙·JSON 스키마를 담은 프롬프트 생성 |
| `buildAssistPrompt(task, type)` | 작업물(이메일/안건/체크리스트…) 생성용 프롬프트 |
| `parseJson(raw)` | LLM 출력에서 코드펜스 제거 후 첫 `{`~마지막 `}` 안전 추출 |
| `normalizePlan(obj)` | 스키마 검증·verdict 보정·**우선순위 정렬·rank 부여** |
| `normalizeArtifact(obj)` | 작업물 정규화 |
| `fallbackPlan(context, now)` | **SDK 없이도** 키워드 규칙으로 결정/타임라인 생성 |
| `fallbackArtifact(task, type)` | 템플릿 기반 작업물 생성 |

### L4. AI 통합 계층

| 기술 | 설명 |
| --- | --- |
| **`@github/copilot-sdk`** | LLM HTTP API가 아니라 **로컬 Copilot CLI 런타임을 JSON-RPC로 제어**하는 SDK. `CopilotClient` → `createSession` → `session.send` → `assistant.message` 수집 → `session.idle`. |
| **`@github/copilot` (런타임)** | SDK가 자식 프로세스로 spawn 하는 실행 파일. `resolveCliPath()`가 node_modules에서 진입점 자동 탐색. |
| **토큰 인증** | `gitHubToken` + `useLoggedInUser:false` → 대화식 로그인 없이 **서버사이드** 세션 인증(클라우드 구동 가능). |
| **모델** | `COPILOT_MODEL=auto` (런타임이 자동 선택). |

### L5. 인프라 / 배포 계층

| 기술 | 설명 |
| --- | --- |
| **Azure App Service (Linux)** | PaaS. 공개 HTTPS URL 자동 제공. |
| **SKU B1 + Always On** | Copilot 런타임(~500MB) 콜드스타트 완화. |
| **Oryx 빌드** | `az webapp up` 시 서버에서 `npm install` 자동 수행. |
| **App Settings** | `COPILOT_GH_TOKEN` 등 시크릿을 **코드 밖**에서 주입(커밋 금지). |
| **TZ 보정** | 호스트가 UTC이므로 `nowKST()`가 `Intl`로 KST 계산. |

### L6. 품질 / 운영 계층

| 기술 | 설명 |
| --- | --- |
| **`scripts/smoke.js`** | 네트워크 비의존 단위 스모크 28종(프롬프트·파서·정렬·폴백 검증). |
| **`judge/` 자체 심사기** | 게이트(P0)·차원 점수·보안 등급을 자동 채점하는 재사용 도구. `npm run judge`. |
| **ADR (`docs/adr/`)** | 5건의 의사결정 기록(브랜치·SDK·피벗·UX·보안). |

## 3. 의도적으로 **쓰지 않은** 것 (Scope 명확화)

> MECE의 "중복 없음"을 위해, 흔히 기대되지만 **일부러 배제**한 기술과 그 이유.

| 미사용 | 이유 |
| --- | --- |
| 프런트 프레임워크(React/Vue) | 빌드·번들 오버헤드 > 이득. 단일 화면이라 Vanilla로 충분. |
| 데이터베이스(SQL/NoSQL) | MVP는 무상태. 영속은 localStorage. |
| 상태관리 라이브러리 | 클라이언트 상태가 단순(plan 1개 + 표시순서). |
| Docker | 환경에 Docker 없음 → App Service 직접 배포로 단순화. |
| 벡터DB / RAG | 과제가 요구하지 않음. 결정 로직은 프롬프트+규칙으로 충분. |

## 4. 의존성 전체 목록 (`package.json`)

```json
"dependencies": {
  "@github/copilot": "^1.0.63",       // Copilot CLI 런타임 (SDK가 spawn)
  "@github/copilot-sdk": "^1.0.2",    // 에이전트 세션 제어 SDK
  "express": "^4.21.2",               // 웹 서버
  "helmet": "^7.2.0"                  // 보안 헤더
}
```

> 런타임 의존성 **단 4개**. "작게 만들어 확실히 동작시킨다"는 설계 철학의 증거.
